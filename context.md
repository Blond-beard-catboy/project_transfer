# Контекст проекта: аналитика и визуализация

## Что сделано
1. **Созданы витрины данных** через Spark-джобу (`jobs/spark_analytics_job.py`):
   - `dm_profit_forecast` – прогноз прибыли по месяцам (в БД `order_db`).
   - `dm_empty_runs` – порожние маршруты (аппенд).
   - `dm_driver_rating` – рейтинг водителей.
2. **Реализован Analytics Service** (порт 8009) с эндпоинтами:
   - `/api/analytics/orders/monthly` – данные для графика.
   - `/api/analytics/routes/popular` – популярные маршруты.
   - `/api/analytics/customers/{customer_id}` – статистика клиента.
3. **Добавлен CORS** в Analytics Service для доступа из браузера.
4. **Создан HTML-дашборд** (`dashboard.html`) с графиком количества заказов.

## Как запустить визуализацию

1. **Запустите все микросервисы** (особенно Analytics Service на порту 8009).
2. **Убедитесь, что Spark-джоба выполнилась** и в БД есть данные (можно проверить через curl).
3. **Запустите простой HTTP-сервер** в папке с `dashboard.html`:
   ```bash
   cd ~/project_transfer
   python3 -m http.server 8081
   ```
4. **Откройте в браузере**: `http://localhost:8081/dashboard.html`
5. **График отобразит количество заказов по месяцам** (в текущих данных – 10 заказов за март 2026).

## Данные из примера (curl)
```bash
curl -H "X-User-ID: 1" -H "X-User-Role: admin" http://localhost:8009/api/analytics/orders/monthly
```
Ответ:
```json
[{"year":2026,"month":3,"total_orders":10,"completed_orders":0,"total_revenue":0.0,"updated_at":"2026-03-17T12:36:09.385137"}]
```

## Если график не отображается
- Проверьте, что CORS добавлен в Analytics Service (файл `main.py`).
- Откройте консоль браузера (F12) – там не должно быть ошибок CORS или 401.
- Обновите страницу без кэша (Ctrl+F5).
