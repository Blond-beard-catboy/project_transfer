# Список микросервисов

| Сервис | Порт | База данных | Назначение |
|--------|------|-------------|------------|
| User Service | 8000 | `user_db` | Регистрация, аутентификация, управление пользователями, расширенный профиль |
| Cargo Service | 8001 | `cargo_db` | CRUD грузов, фильтрация, импорт тестовых данных |
| Route Service | 8002 | `route_db` | Управление маршрутами и точками, добавление грузов, расчёт расстояний |
| Order Service | 8003 | `order_db` | Создание заказов, генерация PDF-договоров, архивация, интеграция с платежами и уведомлениями |
| Notification Service | 8004 | `notification_db` | Эмуляция отправки уведомлений (email/SMS), хранение истории |
| API Gateway | 8005 | — | Единая точка входа, проверка JWT, маршрутизация, проброс заголовков |
| Payment Service | 8006 | `payment_db` | Управление платежами по заказам, имитация оплаты |
| Parser Service | 8007 | `parser_db` | Парсинг тестовых данных о грузах из JSON, импорт в Cargo Service |
| Cart Service | 8008 | `cart_db` | Корзина грузов, массовое оформление заказов |
| Analytics Service | 8009 | `analytics_db` | Агрегированные отчёты (заказы по месяцам, популярные маршруты, статистика клиентов) |

## Подробное описание

### User Service
Отвечает за регистрацию, аутентификацию и управление пользователями. Поддерживает роли `driver`, `dispatcher`, `admin`. Хранит расширенный профиль (компания, ИНН, телефон, адрес, рейтинг, количество заказов). Генерирует JWT-токены с идентификатором пользователя и ролью.

### Cargo Service
Управляет грузами. Позволяет создавать, просматривать, обновлять и удалять грузы. Владелец определяется автоматически из заголовка `X-User-ID`. Поддерживает фильтрацию по статусу, датам и владельцу. Содержит эндпоинт для импорта тестовых грузов из JSON.

### Route Service
Создаёт и управляет маршрутами и их точками. При добавлении груза автоматически создаёт точки погрузки и разгрузки на основе данных груза. Поддерживает изменение статуса точек (выполнена/не выполнена). Имеет заглушку для расчёта расстояний (гаверсинус или константа).

### Order Service
Центральный сервис для заказов. При создании заказа проверяет груз через Cargo Service и создаёт маршрут через Route Service. При подтверждении заказа генерирует PDF-договор, отправляет уведомление через Notification Service и создаёт платёж через Payment Service. Поддерживает архивацию и восстановление заказов.

### Notification Service
Создаёт уведомления (email/SMS) и сохраняет их в базу данных. Вместо реальной отправки просто логирует факт отправки. Может быть расширен асинхронной обработкой через очереди.

### API Gateway
Единая точка входа для клиентов. Проверяет JWT-токены, добавляет заголовки `X-User-ID`, `X-User-Role`, `X-Correlation-ID` и проксирует запросы к соответствующим сервисам. Содержит публичные маршруты для регистрации и логина.

### Payment Service
Управляет платежами для заказов. Создаёт запись о платеже при подтверждении заказа. Позволяет имитировать оплату (эндпоинт `/pay`), меняя статус с `pending` на `paid`. Хранит историю платежей.

### Parser Service
Эмулирует парсер внешних источников данных о грузах. При запуске (`/run`) читает JSON-файл с тестовыми данными и сохраняет их в свою таблицу. Позволяет импортировать выбранный груз в Cargo Service.

### Cart Service
Управляет корзиной грузов для каждого пользователя. Позволяет добавлять, удалять и просматривать грузы. При оформлении заказа (`/checkout`) для каждого груза в корзине создаёт заказ через Order Service и после успеха очищает корзину. Проверяет существование груза через Cargo Service.

### Analytics Service
Агрегирует данные из других сервисов (например, Order Service) и сохраняет их в собственные таблицы для быстрых отчётов. Поддерживает обновление по расписанию (через apscheduler) или по запросу. Предоставляет эндпоинты для получения статистики (заказы по месяцам, популярные маршруты, статистика клиентов).

## Зависимости между сервисами
- Order Service зависит от Cargo, Route, Notification, Payment.
- Route Service зависит от Cargo.
- Cart Service зависит от Cargo и Order.
- Parser Service зависит от Cargo.
- Analytics Service зависит от Order.

Все зависимости реализованы через синхронные HTTP-вызовы с использованием клиентов `httpx`.
