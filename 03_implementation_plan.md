## 3. Пошаговый план реализации (этапы)

### Этап 0: Подготовка окружения
- Установить Docker, docker-compose, Python 3.10+.
- Создать репозиторий со структурой папок (см. ниже).
- Написать `docker-compose.yml` с PostgreSQL (инициализация нескольких баз) и Redis (опционально).
- Создать скрипт для создания нескольких баз в PostgreSQL.

### Этап 1: Разработка шаблона микросервиса
- Создать базовую структуру для сервиса (папки app, core, routers, models, schemas, dependencies).
- Настроить подключение к БД (SQLAlchemy + asyncpg), базовые модели и миграции (Alembic – опционально).
- Реализовать middleware для correlation ID и структурированного логирования.
- Написать общие зависимости (`get_db`, `get_current_user` для User Service).

### Этап 2: User Service
- Реализовать модели и эндпоинты регистрации, логина, получения профиля.
- Добавить JWT-аутентификацию (PyJWT) и хеширование паролей (passlib).
- Написать middleware для проверки токена (использовать в Gateway позже).
- Покрыть тестами (pytest).

### Этап 3: Cargo Service
- Реализовать модели Cargo, эндпоинты CRUD.
- Добавить фильтрацию и поиск (заглушка).
- Реализовать заглушку импорта тестовых грузов из JSON.
- Написать тесты.

### Этап 4: Route Service
- Реализовать модели Route и RoutePoint.
- Эндпоинты для управления маршрутами и точками.
- Добавить заглушку расчёта расстояний.
- Реализовать интеграцию с Cargo Service (HTTP-клиент) для добавления груза.
- Тесты с моками.

### Этап 5: Order Service
- Реализовать модель Order.
- Эндпоинт создания заказа с вызовами к Cargo и Route.
- Эндпоинт изменения статуса с вызовом Notification Service.
- Эндпоинт подтверждения (генерация PDF).
- Интеграционные тесты (моки для Cargo, Route, Notification).

### Этап 6: Notification Service
- Реализовать модель Notification.
- Эндпоинт создания уведомления (эмуляция отправки).
- Опционально: подключить RQ для асинхронной обработки.
- Тесты.

### Этап 7: API Gateway
- Реализовать прокси-сервер на FastAPI.
- Добавить middleware проверки JWT и проброса заголовков.
- Настроить маршрутизацию по префиксам.
- Добавить обработку correlation ID.
- Тесты маршрутизации и проброса.

### Этап 8: Интеграционное тестирование
- Написать сквозные сценарии (регистрация → создание груза → создание заказа → подтверждение → проверка уведомления).
- Запустить все сервисы через docker-compose и выполнить тесты.

### Этап 9: Документирование
- Собрать OpenAPI спецификации (можно через Gateway).
- Написать README, ARCHITECTURE.md (с диаграммой), CHANGELOG.md.
